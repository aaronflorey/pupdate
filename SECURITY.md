# Security Policy

## Reporting a Vulnerability

Use GitHub Private Vulnerability Reporting for sensitive security issues.

- Open the repository on GitHub.
- Go to the Security tab.
- Use `Report a vulnerability` to send a private report.

Do not open a public issue for undisclosed vulnerabilities.

For non-sensitive bugs or hardening suggestions, open a normal GitHub issue.

## Scope

Report issues that could let an attacker:

- execute unintended commands
- expose secrets or local data
- bypass intended safety controls
- trigger unsafe dependency update behavior

Include reproduction steps, impact, affected versions or commits, and any suggested fix if you have one.

## Response

The maintainers will review reports, confirm impact, and coordinate a fix before public disclosure when needed.

This policy documents the reporting path. GitHub Private Vulnerability Reporting must be enabled in the repository settings for private reports to work.
