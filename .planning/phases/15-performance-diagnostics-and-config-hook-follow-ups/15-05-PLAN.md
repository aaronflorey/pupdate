# Plan 15-05 - Expand user config beyond root_directories

## Objective

Broaden the user configuration surface beyond `root_directories` so operators can control more of `pupdate` behavior without shell-script patching or code changes.

## Scope

- Define the next small set of config options that provide clear user value.
- Keep config growth consistent with the existing config-loading model and current defaults behavior.
- Avoid speculative schema expansion that is not backed by an approved behavior need.

## Tasks

1. Identify the most valuable next config knobs based on currently hard-coded or flag-only behavior.
2. Define the minimal schema, defaults, validation, and docs changes needed to support them cleanly.
3. Add config parsing and behavior coverage that proves new options interact correctly with existing defaults.

## Verification

- Config-focused tests for defaults, explicit settings, and invalid values.
- Documentation review confirming the expanded config surface is accurately described.

## Done When

- Users can configure more than `root_directories` through the supported config file path.
- The expanded config behavior is documented and tested without breaking current defaults.
