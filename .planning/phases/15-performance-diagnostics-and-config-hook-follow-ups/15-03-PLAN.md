# Plan 15-03 - Prune stale state entries

## Objective

Keep `.pupdate` truthful over time by pruning stale entries when tracked directories, ecosystems, or lockfiles disappear.

## Scope

- Focus on stale-state cleanup semantics in the existing state file.
- Preserve entries that are still relevant to active tracked projects.
- Avoid unrelated state-schema changes unless needed to support safe pruning.

## Tasks

1. Identify the stale-entry cases that can accumulate today when repos or tracked ecosystems disappear.
2. Define when pruning should happen and how it should interact with normal run and skip flows.
3. Add regression coverage for removed directories, removed lockfiles, and partial ecosystem disappearance cases.

## Verification

- State-focused tests covering safe retention and safe removal behavior.
- Manual review of `.pupdate` lifecycle semantics after representative disappearance scenarios.

## Done When

- Stale entries no longer persist indefinitely after tracked targets disappear.
- Pruning behavior is predictable and does not remove still-valid state.
