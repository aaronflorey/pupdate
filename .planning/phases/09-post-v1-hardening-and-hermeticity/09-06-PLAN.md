# Plan 09-06 - Treat missing user config as defaults

## Objective

Stop creating user config files during `run` and instead treat a missing config file as implicit default values.

## Scope

- Remove auto-create-on-run behavior from config-loading call sites.
- Keep default config values the same when the file is absent.
- Update config-related tests and any status/config command behavior affected by the missing-file contract.

## Tasks

1. Trace where config creation is triggered today and reduce it to explicit user-facing flows only, if any remain necessary.
2. Update runtime config loading to return default values when the user config file is missing.
3. Rewrite command tests to assert no config file is created during `run` while preserving current default behavior.

## Verification

- `go test ./cmd/pupdate -count=1`
- `go test ./... -count=1`

## Done When

- `pupdate run` no longer creates a config file as a side effect.
- Missing user config behaves exactly like default settings.
