# Plan 10-01 - Make `root_directories` matching filesystem-aware

## Objective

Make configured `root_directories` matching respect case-sensitive filesystem behavior instead of always lowercasing paths before comparison.

## Scope

- Audit the `cmd/pupdate/config.go` normalization path used by `isTopLevelDirectoryWithinRoot`.
- Remove or narrow unconditional lowercasing so path comparisons reflect the actual resolved directory names.
- Update regression coverage in `cmd/pupdate` for case-sensitive root matching while preserving current `~` expansion and top-level-only semantics.

## Tasks

1. Confirm the minimal comparison change needed in `cmd/pupdate/config.go` to stop forcing case-insensitive root matching.
2. Update the affected `cmd/pupdate` tests so mixed-case configured roots only match when the real on-disk path matches.
3. Re-run focused command-package tests covering allowed roots, denied roots, `~`, and mixed-case inputs.

## Verification

- `go test ./cmd/pupdate -count=1`

## Done When

- `root_directories` matching no longer succeeds solely because both paths were lowercased.
- Regression tests encode the intended behavior for case-sensitive filesystems.
