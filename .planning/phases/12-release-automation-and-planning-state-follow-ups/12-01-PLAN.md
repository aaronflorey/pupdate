# Plan 12-01 - Reconcile duplicate release automation workflows

## Objective

Remove or reconcile overlapping release automation workflows so releases follow one reliable GitHub Actions path and Homebrew publishing uses the token contract already required by `.goreleaser.yaml`.

## Scope

- Audit `.github/workflows/release.yaml`, `.github/workflows/release-please.yml`, `.goreleaser.yaml`, and any nearby release docs or validation artifacts.
- Choose one supported release automation path consistent with the intended Release Please plus GoReleaser flow.
- Update workflow wiring so the surviving path passes `HOMEBREW_TAP_GITHUB_TOKEN` when GoReleaser publishes the Homebrew tap.

## Tasks

1. Confirm exactly how the two current release workflows overlap and what behavior differs between them.
2. Implement the smallest workflow and documentation change set that leaves one auditable release path.
3. Verify the surviving release path still matches `.goreleaser.yaml` requirements, especially Homebrew token wiring.

## Verification

- Manual review of the final workflow/config combination showing a single supported release path.
- Manual review that the GoReleaser job exposes `HOMEBREW_TAP_GITHUB_TOKEN` to satisfy `.goreleaser.yaml` brew publishing.

## Done When

- Only one supported release automation path remains in repository workflow configuration.
- The surviving GoReleaser workflow wiring matches the Homebrew token contract declared in `.goreleaser.yaml`.
