# Plan 13-01 - Update release-planning documentation and state text

## Objective

Update the remaining release-planning documentation and planning-state text so every reference matches the surviving `.github/workflows/release.yaml` file and its current push-to-`main`/`master` plus conditional created-tag checkout model.

## Scope

- Audit release-related planning and state artifacts for references to `.github/workflows/release.yml`, removed duplicate-workflow wording that no longer fits, or stale release-flow descriptions.
- Update those artifacts to reference `.github/workflows/release.yaml` consistently.
- Align the documented release flow with the current workflow behavior without changing workflow YAML.

## Tasks

1. Identify each release-planning or state artifact that still references the old workflow filename or outdated release-flow wording.
2. Update the narrowest set of artifacts so all surviving references describe the current `release.yaml` path and behavior consistently.
3. Re-check the edited planning/state text for filename, trigger-scope, and created-tag checkout accuracy.

## Verification

- Manual review that updated release-planning and state artifacts consistently reference `.github/workflows/release.yaml`.
- Manual review that the documented flow matches the current workflow triggers and conditional checkout of `needs.release-please.outputs.tag_name`.

## Done When

- No edited release-planning or state artifact still refers to `.github/workflows/release.yml` or an outdated release-flow model.
- The surviving release documentation describes push-to-`main`/`master` triggering plus conditional created-tag checkout accurately.
