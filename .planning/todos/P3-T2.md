# P3-T2: Apply workspace_globs to detection

Status: complete
Type: implementation
Phase: 3 monorepo scan expansion
Depends on: P3-T1
PRD refs: `.planning/update.md` -> `Product Gaps` / `2. Revisit detection depth for common monorepo layouts`
Approved-plan refs: `.planning/quick/phase-15-performance-diagnostics-config-hook-planning.md`

## Objective

Apply `workspace_globs` during detection while keeping the current shallow default scan unchanged.

## Context

- The default traversal contract is a performance constraint and must remain unchanged when `workspace_globs` is unset.
- The new behavior should only expand candidate directories matched by approved config patterns.
- Detection changes must preserve existing freshness, manager, and package-root semantics.

## Expected files

Create or modify only these files unless a stop condition applies:

- `internal/detection/detector.go`: add opt-in expansion of candidate directories from configured workspace globs.
- `internal/detection/detector_test.go`: add cases for configured glob matches and unchanged default behavior.
- `internal/detection/detector_benchmark_test.go`: extend or inspect guardrail coverage if the new behavior changes benchmark setup.
- `cmd/pupdate/run_execution.go`: thread resolved `workspace_globs` into detection.
- `cmd/pupdate/status.go`: thread resolved `workspace_globs` into detection for status.

## Required changes

1. Extend detection inputs so the resolved `workspace_globs` config reaches the detector.
2. Expand candidate scanning only for directories matched by the configured globs, while preserving the current default scan exactly when the setting is absent.
3. Add tests that prove common layouts such as `apps/*` or `services/*` can be discovered only when explicitly configured.

## Non-goals

- Do not change the default traversal depth for users who do not configure `workspace_globs`.
- Do not update README docs in this task.

## Acceptance criteria

- Repositories without `workspace_globs` behave exactly as they do today.
- Configured glob patterns can include additional monorepo roots such as `apps/*` without broad recursive scanning everywhere.
- Detection tests cover both the unchanged default path and the configured opt-in path.

## Checks

Run or inspect:

- `go test ./internal/detection ./cmd/pupdate -count=1`
- `go test ./internal/detection -run '^$' -bench 'BenchmarkDetectProjectTree' -count=1`

## Review instructions

The reviewer must verify:

- The implementation matches this brief and does not add unrequested behavior.
- All acceptance criteria are satisfied.
- Required tests or checks pass, or failures are explained and unrelated.
- No later-phase work was pulled into this task.

## Stop conditions

Stop and ask the orchestrator or user if:

- The implementation would require recursive scanning beyond the approved glob model.
- Benchmark results indicate a default-path regression that needs a new design decision.
- The work would require changing files outside the allowed list.
