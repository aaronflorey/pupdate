## Objective

Pin the development Go version in `mise.toml` to match `go.mod` and contributor guidance.

## Context

- The PRD calls out drift risk from `mise.toml` using `go = "latest"` while `go.mod` pins the project toolchain.
- The repo already documents Go 1.26 in `go.mod` and `CONTRIBUTING.md`.
- This should remain a small tooling-alignment change, not a broader toolchain policy rewrite.

## Expected files

Create or modify only these files unless a stop condition applies:

- `mise.toml`: pin the Go tool version to the project-supported version.
- `CONTRIBUTING.md`: adjust wording only if it needs to match the pinned `mise` guidance.

## Required changes

1. Replace the floating Go tool declaration in `mise.toml` with the pinned project version.
2. Confirm contributor guidance matches the pinned version.
3. Keep the scope limited to toolchain alignment for Go.

## Non-goals

- Do not change `go.mod` in this task.
- Do not add new tools or task definitions to `mise.toml`.

## Acceptance criteria

- `mise.toml` no longer uses `latest` for Go.
- The pinned version aligns with `go.mod`.
- Contributor docs remain consistent with the pinned toolchain guidance.

## Checks

Run or inspect:

- Manual inspection of `mise.toml`, `go.mod`, and `CONTRIBUTING.md`

## Review instructions

The reviewer must verify:

- The implementation matches this brief and does not add unrequested behavior.
- All acceptance criteria are satisfied.
- Required tests or checks pass, or failures are explained and unrelated.
- No later-phase work was pulled into this task.

## Stop conditions

Stop and ask the orchestrator or user if:

- The repo uses a different Go patch version policy than `go.mod` alone communicates.
- The work would require changing CI setup or release configuration beyond the allowed files.
- Pinning the version would conflict with an existing documented contributor workflow.
