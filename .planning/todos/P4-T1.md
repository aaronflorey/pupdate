# P4-T1: Expand installation guidance

Status: pending
Type: implementation
Phase: 4 docs, tooling, and CLI confidence
Depends on: P1-R1
PRD refs: `.planning/update.md` -> `UX and Adoption` / `7. Expand installation guidance in the README`
Approved-plan refs: none

## Objective

Expand the README install section to support Homebrew and `go install` alongside the existing `bin` workflow.

## Context

- The PRD calls out adoption friction from documenting `bin` only.
- The user chose to document Homebrew and `go install` as intended supported paths.
- GoReleaser already contains Homebrew publishing configuration, so the docs should align with the shipped release story.

## Expected files

Create or modify only these files unless a stop condition applies:

- `README.md`: expand the install section and clarify the available install paths.

## Required changes

1. Add install instructions for Homebrew.
2. Add install instructions for `go install`.
3. Keep the existing `bin` workflow documented in a way that does not conflict with the new supported paths.

## Non-goals

- Do not change release automation or package publishing behavior.
- Do not modify unrelated README sections in this task.

## Acceptance criteria

- README contains concrete Homebrew and `go install` install paths.
- README still documents `bin` without implying it is the only supported path.
- The install section stays consistent with the current release configuration.

## Checks

Run or inspect:

- Manual inspection of README install steps against `.goreleaser.yaml`

## Review instructions

The reviewer must verify:

- The implementation matches this brief and does not add unrequested behavior.
- All acceptance criteria are satisfied.
- Required tests or checks pass, or failures are explained and unrelated.
- No later-phase work was pulled into this task.

## Stop conditions

Stop and ask the orchestrator or user if:

- The exact Homebrew formula or `go install` version string cannot be stated truthfully from the current repo state.
- The docs would require unapproved release-process changes.
- The work would require editing files outside the allowed list.
