# Plan 09-02 - Make `cmd/pupdate` tests hermetic

## Objective

Remove ambient user-config coupling from command tests so `cmd/pupdate` regression coverage is deterministic and does not depend on side effects in the caller's config directory model.

## Scope

- Isolate config-path and config-loading behavior behind testable seams already used by the command package.
- Replace tests that currently depend on config auto-creation with assertions about explicit defaults.
- Keep user-visible command behavior unchanged except where later Plan 09-06 intentionally removes auto-create-on-run behavior.

## Tasks

1. Identify the current process-global config helpers used by `cmd/pupdate` tests and where they leak ambient assumptions.
2. Introduce the smallest injection or helper pattern needed for hermetic command tests.
3. Rewrite affected tests to assert defaults and config-loading behavior without depending on filesystem state outside the test sandbox.

## Verification

- `go test ./cmd/pupdate -count=1`

## Done When

- `cmd/pupdate` tests run hermetically with only test-owned config inputs.
- Ambient config resolution is no longer required to prove command behavior.
