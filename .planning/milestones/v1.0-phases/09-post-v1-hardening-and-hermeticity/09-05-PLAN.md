# Plan 09-05 - Harden state-file persistence

## Objective

Increase `.pupdate` durability guarantees by fsyncing the parent directory after atomically replacing the state file.

## Scope

- Extend `internal/state.Store.Save` to sync directory metadata after rename where supported.
- Keep the existing temp-file write, file sync, and atomic rename flow intact.
- Add tests around the new durability step and failure propagation.

## Tasks

1. Add parent-directory sync after `os.Rename` in the state store save path.
2. Handle unsupported directory-sync environments in the narrowest safe way.
3. Add unit coverage for successful saves and injected sync failures.

## Verification

- `go test ./internal/state -count=1`

## Done When

- State saves include a parent-directory durability step after rename.
- Tests cover the new save-path behavior without weakening atomic-write semantics.
