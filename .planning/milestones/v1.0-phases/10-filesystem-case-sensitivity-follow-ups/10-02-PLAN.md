# Plan 10-02 - Preserve matched lockfile paths through detection and freshness

## Objective

Preserve the actual on-disk matched lockfile path through detection and freshness so mixed-case lockfiles found case-insensitively still work on case-sensitive filesystems.

## Scope

- Audit the detection path contract in `internal/detection` and the file hashing/state-key flow in `internal/freshness`.
- Preserve real matched relative paths while keeping ecosystem identification stable.
- Keep compatibility adjustments as small as possible, especially where existing lowercase state keys may already exist in `.pupdate` data.

1. Confirm where canonical signal names replace actual matched filenames in detection results.
2. Implement the minimal change so freshness stats and hashes the real matched path while still comparing against prior state correctly.
3. Add regression coverage for mixed-case lockfiles discovered case-insensitively and then evaluated on a case-sensitive filesystem.

## Verification
